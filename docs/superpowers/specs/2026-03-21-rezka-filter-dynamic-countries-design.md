# Дизайн-спецификация: Накопительный список стран с сортировкой

**Дата:** 2026-03-21
**Статус:** На утверждении

## 1. Обзор
Доработка логики формирования и отображения списка стран в панели фильтров расширения Rezka Filter. Цель — сделать список стран накапливаемым (глобальным для всех сессий), чтобы однажды обнаруженная страна оставалась доступной для управления, а также улучшить UX путем сортировки списка.

## 2. Функциональные изменения

### Накопительный глобальный список стран (`storage.js`)
- Вводится новая сущность в хранилище: `knownCountries` (массив строк).
- **Логика пополнения:**
  - При каждом сканировании страницы парсер собирает уникальные страны с карточек на экране.
  - Модуль хранилища сравнивает этот список со своим массивом `knownCountries` (а также с `blacklistedCountries`, чтобы старые фильтры не пропадали).
  - Все новые страны добавляются в `knownCountries`.
  - Измененный массив сразу сохраняется в `chrome.storage.local`.

### Улучшенная сортировка (`ui.js`)
- При отрисовке панели список `knownCountries` разделяется на две логические группы:
  1. **Заблокированные страны** (присутствуют в `blacklistedCountries`).
  2. **Остальные страны** (отсутствуют в `blacklistedCountries`).
- Каждая группа сортируется по алфавиту внутри себя.
- При выводе в DOM сначала добавляются чекбоксы из группы "Заблокированные", затем чекбоксы из группы "Остальные".
- Это гарантирует, что выбранные пользователем фильтры всегда находятся в верхней части списка, не требуя прокрутки.

## 3. Взаимодействие модулей
1. **`parser.js`:** Без изменений (отдает массив стран, найденных на текущем экране).
2. **`main.js` (`refreshState`):** 
   - Запрашивает страны у парсера (`rfParser.getAllCountriesOnPage()`).
   - Передает найденные страны в хранилище для пополнения словаря (`rfStorage.addKnownCountries(...)`).
   - Передает полный словарь `rfStorage.knownCountries` в UI для отрисовки панели.
3. **`storage.js`:** 
   - Добавляется поле `knownCountries: []`.
   - Добавляется метод `addKnownCountries(newCountries)`.
   - В метод `load()` добавляется чтение `knownCountries` (с объединением с `blacklistedCountries` для совместимости).
   - В метод `save()` добавляется сохранение `knownCountries`.
4. **`ui.js`:**
   - Метод `createOrUpdatePanel` принимает полный список стран.
   - Перед созданием элементов `label` и `input` список сортируется: сначала элементы, входящие в `blacklistedCountries` (алфавитно), затем остальные (алфавитно).

## 4. Обработка краевых случаев
- **Сброс/Очистка кэша:** При переустановке расширения или очистке данных сайтов браузером `knownCountries` сбросится, но это ожидаемое поведение.
- **Миграция:** У существующих пользователей `knownCountries` при первом запуске будет равен пустому массиву `[]` или `undefined`. При первой загрузке мы объединим уже существующий `blacklistedCountries` с `knownCountries`, чтобы старые заблокированные страны сразу появились в списке наверху.
